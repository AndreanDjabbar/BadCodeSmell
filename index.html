Bad Code Smell

Bad Code Smell adalah sebuah indikator pada code atau tanda-tanda pada code yang menunjukan adanya masalah di dalamnya. Biasanya ditandai dengan adanya code yang memiliki duplikasi, pembuatan fungsi yang terlalu panjang, ketergantungan dalam code, dan masih banyak yang lainnya. Bad Code Smell juga dikelompokan ke dalam 5 jenis seperti: The Bloaters, Object Oriented Abuser, Change Preventer, The Dispensible, dan The Couplers.

Jika hal seperti ini dibiarkan, maka akan berpengaruh pada code kita. Berikut ini adalah contoh dari pengaruh Bad Code Smell terhadap code atau program kita:

  • Sulitnya dalam melakukan maintenance atau pemeliharaan code kita. Akan sulit jadinya jika kita ingin menambahkan sebuah fitur baru, memperbaiki bug, ataupun melakukan perubahan lainnya jika terdapat Bad Code Smell.
  • Kinerja yang buruk. Bad Code Smell juga akan membuat kinerja program yang buruk karena adanya code yang berlebihan.
  • Sulitnya dalam melakukan kolaborasi. Tim developer akan mengalami kesulitan dalam melakukan kolaborasi karena adanya ketidakjelasan atau keambiguan yang ada pada code, akibat dari Bad Code Smell.
  • Sulitnya dalam melakukan Testing/Uji Coba. Bad Code Smell juga akan menyulitkan kita jika hendak melakukan pengujian atau testing code karena akan sulit dalam melakukan pengisolasian code.

Lalu bagaimana solusi atau penanganan terhadap masalah Bad Code Smell ini?

Jawabannya adalah Refactoring Code.

Refactoring code adalah upaya untuk Membuat sebuah optimisasi ataupun perubahan pada sebuah struktur software tanpa mengubah fungsi software tersebut. Simpelnya adalah membuat struktur software lebih mudah dipahami. Sehingga, nantinya akan mudah untuk melakukan perubahan pada code tersebut ataupun menemukan sumber permasalahan jika terjadi kesalahan pada software tersebut.
